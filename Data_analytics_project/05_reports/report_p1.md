The objective of this practical is to prepare the dataset and workspace for Exploratory Data Analysis (EDA).

## Observations

### 1. What is one row in this dataset representing?

One row represents one sales transaction/order record of a customer.


### 2. Which columns look numerical, categorical and date/time?

Numerical columns:
- Sales
- Quantity
- Discount
- Profit

Categorical columns:
- Category
- Region
- Segment
- Ship Mode

Date/Time columns:
- Order Date
- Ship Date


### 3. What can go wrong if a team starts analysis without a data dictionary?

Without a data dictionary:
- Columns can be misunderstood.
- Wrong analysis may be performed.
- Data quality issues may be ignored.


## Conclusion

Creating documentation and organizing files properly improves the data analysis workflow and helps teams understand the dataset easily.
